==============Over Loading===========
*java10_ Over Loaing
- DB관련해서 사용용도
- 항상 super(); -> this() 순서로 사야외어야함
- 상속 public / static / final

*java11
- Method Overloading
rule- = 이름, 인자값, r-t
통일적인 metod로 단일한 관리(인자값)

형태
::

부모 클래스

Employee e = new Employee("111-111", "아이유", new MyDate(1980,1,3), 300000.0);
Employee e1 = new Employee("777-777");


자식 클래스

public Employee(String ssn, String name, MyDate birthDate, double salary) {
		// super(); 부모의 기본 생성자는 무조건 호출됨.
		this.ssn = ssn;
		this.name = name;
		this.birthDate = birthDate;
		this.salary = salary;
	}
public Employee(String ssn) {
		this.ssn = ssn;
		}


사용이유
::

ex) 회원가입시 정보를 정상적으로는 정보를 5개를 입력해야하는데 어떤 회원은 2개만 입력을 하였다.
이 때, 이 회원은 돈을 지불했기때문에 DB에 회원정보를 올려야하므로,
정상적인 명시적생성자 말고도, 입력이 가능하게 이보다 더 짧은 정보를 가진 명시적생성자를 생성하기위해
Over Loading 하는 것이다.
